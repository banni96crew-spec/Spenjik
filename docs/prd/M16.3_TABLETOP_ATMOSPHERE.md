## 1. Purpose

M16.3 turns the existing functional tabletop layout and card visual system into a cohesive dark tabletop atmosphere.

The game should feel like it is being played on a physical table inside a mafia-owned underground casino in a criminal city.

The mood may be inspired by the physical, oppressive, tactile tabletop feeling of games like Inscryption, but must not copy its IP, characters, exact symbols, story, camera language or distinctive proprietary assets.

M16.3 exists to:

* remove remaining default Godot/prototype visual feel;
* establish a consistent dark tabletop mood;
* make the table surface, panels, cards and side information feel like one physical scene;
* introduce a mafia casino / criminal city atmosphere through UI presentation;
* make the UI feel dirty, worn, secretive and high-stakes;
* preserve readability and usability at `1280x720` and `1920x1080`;
* keep all gameplay interaction through `GameStateManager`;
* preserve deterministic replay and all existing gameplay tests.

M16.3 must not implement new gameplay rules.

## 4. Runtime State

### 4.1. Atmosphere State Rule

M16.3 may not introduce authoritative gameplay state.

Allowed UI-local atmosphere state:

* current visual screen background state;
* current theme variant if purely presentation-only;
* hover/selected visuals already owned by UI;
* temporary visual error emphasis;
* local expanded/collapsed panel state already allowed by UI specs.

Forbidden atmosphere state:

* player Nal;
* player VP;
* player hand contents;
* market contents;
* contract progress;
* contact ownership;
* debt state;
* random step;
* phase state;
* combat result;
* AI decision state.

Gameplay state must come from:

```text
GameStateManager.get_view()
GameStateManager.get_state_snapshot()
GameStateManager public selectors
```

UI must not read:

```text
GameStateManager.state
```

## 5. Atmosphere Direction

### 5.1. High-level Mood

The UI should feel like:

* a private back-room table;
* an illegal casino room;
* a mafia-controlled game;
* money moving under the table;
* smoke, dust, old paper, dirty cards;
* a city where every district is being fought over;
* criminal bureaucracy: ledgers, contracts, debts, protection;
* expensive but decayed;
* tense, quiet and dangerous.

The UI should not feel like:

* default Godot UI;
* clean spreadsheet/dashboard;
* mobile card battler;
* fantasy tavern;
* neon sci-fi casino;
* cartoon mafia parody;
* bright arcade board game;
* web app.

### 5.2. Inscryption-like Inspiration Boundary

Allowed inspiration:

* physical tabletop feeling;
* dark room pressure;
* tactile card presence;
* minimal but oppressive table scene;
* moody lighting;
* strong silhouettes;
* worn physical objects;
* readable but unsettling UI.

Forbidden copying:

* no direct Inscryption symbols;
* no copied board layout;
* no copied characters;
* no copied lore;
* no copied card frames;
* no copied typography if proprietary;
* no copied puzzles, masks, scales or eye/camera motifs;
* no copied audio/animation language.

The target is not “make Inscryption”.
The target is “make The Turf feel like a mafia-owned criminal tabletop game”.

## 6. Visual Pillars

### 6.1. Dirty Luxury

The casino should feel controlled by money, but not clean.

Visual traits:

* worn black surfaces;
* dirty gold or warm off-white accents;
* old velvet/table felt feel;
* stained paper;
* scratched metal;
* dim expensive lamps implied by color;
* faded elegance.

Avoid:

* shiny gold overload;
* clean luxury hotel look;
* bright casino floor look.

### 6.2. Criminal Administration

The game is about districts, contracts, debts and power.

Visual traits:

* case-file side column;
* paper/ledger styling;
* stamped labels;
* muted warning red;
* document-like panel headers;
* official-looking but corrupt UI details.

Avoid:

* modern SaaS dashboard;
* police evidence board as the main identity;
* clean bureaucratic forms.

### 6.3. Tabletop Physicality

The screen should feel like cards and panels are placed on a table.

Visual traits:

* table surface behind central gameplay;
* panels that feel like cards, papers or boards;
* subtle borders and shadows;
* worn separators;
* consistent material language.

Avoid:

* floating flat rectangles;
* unrelated UI boxes;
* pure black empty void with no surface identity.

### 6.4. Threat and Pressure

The game should feel tense.

Visual traits:

* dark negative space;
* narrow color palette;
* controlled red accents for danger/errors;
* defensive badges and warnings visible but not flashy;
* low-contrast grime that does not reduce readability.

Avoid:

* horror gore;
* excessive blood;
* bright red everywhere;
* comedy gangster clichés.

## 7. Palette Rules

### 7.1. Required Palette Direction

Primary colors:

* near-black;
* dark charcoal;
* dirty brown-black;
* muted casino felt green if used very subtly;
* dirty ivory / warm off-white text;
* muted gray;
* dark desaturated red for danger/accent;
* muted brass/gold only for restrained casino/luxury accents.

### 7.2. Forbidden Palette Direction

Do not use:

* default Godot blue/gray as primary UI identity;
* bright saturated green;
* bright fantasy gold;
* neon cyan/purple/pink;
* clean white large panels;
* mobile-game gradients;
* large bright red surfaces.

### 7.3. Contrast Rule

Dark atmosphere must not break readability.

Required:

* body text readable at `1280x720`;
* important numbers readable;
* disabled state distinguishable from enabled state;
* errors readable;
* selected state visible.

## 8. Surface and Material Rules

### 8.1. Table Surface

The central table surface should visually support:

* market cards;
* action panel;
* Street Deal panel;
* human zone;
* AI zones.

Allowed implementations:

* ColorRect with layered StyleBoxes;
* TextureRect with project-local valid texture;
* procedural dark surface using StyleBoxFlat;
* subtle grime/noise texture;
* table-felt or worn tabletop impression.

Forbidden implementations:

* runtime loading from absolute `C:\...` paths;
* malformed reference image imports;
* huge uncompressed texture without reason;
* photo-realistic image that makes UI unreadable;
* background that hides cards/buttons.

### 8.2. Panel Surface

Panels should feel like:

* paper folders;
* card stock;
* casino table plaques;
* old ledgers;
* worn UI boards.

Panels must not look like:

* default Godot panels;
* clean Windows forms;
* modern web cards;
* debug rectangles.

### 8.3. Border Rule

Borders should be:

* subtle;
* worn or muted;
* consistent across panels;
* visible enough to separate zones.

Avoid thick bright borders unless used for selected/danger state.

### 8.4. Shadow/Depth Rule

M16.3 may use light presentation-only shadow/depth.

Allowed:

* subtle StyleBox shadow-like dark margins;
* layered panels;
* dark outlines.

Forbidden:

* heavy animated shadows;
* gameplay-driven lighting;
* shader complexity that risks breaking tests/import;
* visual effects owned by later animation milestones.

## 10. Scene Atmosphere Rules

### 10.1. `Main`

`Main` may apply the global theme and root background.

`Main` must not:

* own gameplay state;
* call owner gameplay modules;
* perform phase orchestration;
* load external absolute assets.

### 10.2. `GameRoot`

`GameRoot` may own:

* screen-level atmosphere surface;
* root theme assignment;
* background layering;
* global error display style.

`GameRoot` must not:

* mutate gameplay state;
* compute gameplay values;
* bypass `GameStateManager`.

### 10.3. `GameScreen`

`GameScreen` should feel like the main table.

Required atmosphere responsibilities:

* table surface behind central gameplay;
* AI/Human zones visually placed around the table;
* side info column visually like case file/ledger;
* center table not empty black void;
* phase header integrated into the tabletop scene.

### 10.4. `SetupScreen`

Setup should feel like entering the table, not a plain form.

Allowed visual direction:

* seed as case/file entry;
* role selection as dirty cards/tabs;
* contract offers as papers/cards.

Forbidden:

* changing setup order;
* changing contract generation;
* adding gameplay tutorial logic.

### 10.5. `GameOverScreen`

Game Over should feel like the end of a criminal table session.

Allowed visual direction:

* final ledger/result paper;
* winner stamped/marked;
* dark table still visible.

Forbidden:

* recomputing winner;
* changing final result fields.
