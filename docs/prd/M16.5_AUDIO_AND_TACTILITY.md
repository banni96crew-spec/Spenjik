## 1. Purpose

M16.5 adds UI-only audio and tactile feedback to the existing tabletop UI.

The game should feel more physical and responsive through sound:

* card hover;
* card select;
* button click;
* denied action;
* purchase;
* attack;
* block/defense;
* discard;
* phase transition;
* Street Deal choice;
* contact selection;
* contract claim;
* Game Over reveal;
* low dirty ambient loop.

M16.5 exists to:

* make the table feel physical;
* reinforce player actions with short sounds;
* support the mafia-owned underground casino mood;
* make denied actions easier to understand;
* make combat/block/purchase outcomes more satisfying;
* add ambient pressure without distracting from gameplay;
* preserve `GameStateManager` as the only gameplay-facing UI API;
* preserve deterministic replay and all existing gameplay tests.

M16.5 must not implement new gameplay rules.

## 3. Core Terms

| Term             | Meaning                                                                                  |
| ---------------- | ---------------------------------------------------------------------------------------- |
| Audio Feedback   | Short sound played after UI action, facade result or UI state change.                    |
| Tactility        | Feeling of physical interaction created through short sound, visual feedback and timing. |
| SFX              | Short one-shot sound effect.                                                             |
| Ambience         | Low continuous background sound loop.                                                    |
| Audio Event      | Stable UI-level event name such as `card_hover`, `purchase_success`, `attack_blocked`.   |
| Audio Bus        | Godot audio bus used for grouping UI SFX or ambience.                                    |
| Audio Controller | Optional UI-only node/script that maps UI events to AudioStreamPlayers.                  |
| Gameplay State   | Authoritative runtime state owned by `GameStateManager`.                                 |

## 4. Runtime State

### 4.1. Audio State Rule

M16.5 may store temporary UI-only audio state.

Allowed UI-local audio state:

* SFX volume;
* ambience volume;
* muted/unmuted;
* last played UI event;
* local cooldown for hover spam;
* local round-robin index for sound variation;
* active ambient stream reference;
* active one-shot stream reference.

Forbidden audio gameplay state:

* player Nal;
* player VP;
* player hand contents;
* market contents;
* contract progress;
* contact ownership;
* debt state;
* random step;
* phase state as authority;
* combat result as authority;
* AI decision state.

Audio must not write gameplay state.

Audio must not read:

```text
GameStateManager.state
```

Audio may react to:

```text
GameStateManager.state_changed
GameStateManager.action_failed
GameStateManager.phase_changed
GameStateManager.game_started
GameStateManager.game_ended
UI-local panel/widget signals
facade command result dictionaries passed by UI owner
```

## 5. Audio Direction

### 5.1. High-level Feel

Audio should feel like:

* cards sliding on dirty table felt;
* old casino chips;
* muted metal clicks;
* paper/cardboard movement;
* low room ambience;
* distant criminal city pressure;
* controlled danger;
* mafia back-room silence.

Audio should not feel like:

* arcade casino slot machine;
* mobile game reward spam;
* fantasy magic;
* sci-fi UI beeps;
* bright cartoon sounds;
* loud jump scares;
* comedy gangster clichés;
* clean corporate UI clicks.

### 5.2. Mafia Underground Casino Mood

The sound world should suggest:

* private illegal club;
* hidden gambling table;
* debt and danger;
* old ventilation;
* distant city noise;
* muffled voices outside the room;
* cards and chips handled quietly;
* tense silence between actions.

Avoid direct copyrighted samples, recognizable movie/game audio, or external licensed material without explicit license.

## 7. Sound Selection Rules

### 7.1. No Gameplay Random Rule

Audio must not consume gameplay random.

Forbidden:

```text
SeededRandom
SeededPicker
state["random"]
randf()
randi()
randomize()
RandomNumberGenerator
```

Audio variation may use:

* fixed one-shot sound per event;
* deterministic UI-local round-robin list;
* manually assigned variants without random;
* stable event-to-stream mapping.

If pitch variation is implemented, it must be deterministic and UI-local, not gameplay-random.

### 7.2. No Gameplay Logic Rule

Audio must not compute gameplay outcomes.

Audio may play sounds only from:

* facade command result success/failure;
* facade signals;
* already visible view/log information;
* UI-local interaction events.

Audio must not call:

* `MarketLogic`;
* `CombatEngine`;
* `ContractLogic`;
* `ContactLogic`;
* `StreetDealLogic`;
* `AIBotController`;
* `PriceLogic`.

### 7.3. Failure Sound Rule

Failed command should play `ui_denied` only after facade failure or disabled UI attempt.

Do not play success sound before facade success.

### 7.4. Success Sound Rule

Success sound must be tied to confirmed result.

Examples:

* play `purchase_success` only if `GameStateManager.buy_card(...)` returned `ok == true`;
* play `contract_claim` only if `claim_contract` returned `ok == true`;
* play `ui_denied` if result returned `ok == false`.

## 8. Audio Asset Rules

### 8.1. Project-local Paths

All runtime audio assets must be project-local:

```text
res://assets/audio/sfx/
res://assets/audio/ambience/
```

Allowed examples:

```text
res://assets/audio/sfx/ui_click.ogg
res://assets/audio/sfx/card_select.ogg
res://assets/audio/sfx/purchase_success.ogg
res://assets/audio/sfx/attack_blocked.ogg
res://assets/audio/ambience/underground_room_loop.ogg
```

Forbidden runtime paths:

```text
C:\Users\
CARD_STYLE_REFERENCE
http://
https://
user://
```

### 8.2. Audio Format

Preferred formats:

* `.ogg` for loops and compressed SFX;
* `.wav` only for very small short SFX if justified.

Avoid:

* large uncompressed WAV files;
* malformed audio extension;
* external absolute paths;
* runtime download;
* copyrighted files without license.

### 8.3. Asset Size Rule

Audio assets should be small and practical.

Guidelines:

* short UI SFX should usually be under 1 second;
* ambience loop should be reasonably compressed;
* no huge audio assets without explicit reason.

### 8.4. License Rule

Audio assets must be:

* generated by the user/project;
* original;
* properly licensed;
* public domain / CC0 / allowed commercial use;
* or placeholders created only for development.

Do not include copyrighted audio ripped from other games, movies or sound libraries.

## 9. Audio Bus Rules

### 9.1. Required Bus Groups

M16.5 should support at least:

```text
Master
UI
Ambience
```

If bus setup is not practical through code/tests, use safe scene-level volume properties and document the limitation.

### 9.2. Volume Defaults

Recommended default behavior:

* UI SFX audible but not loud;
* ambience quieter than SFX;
* denied action clear but not harsh;
* Game Over sound stronger but not excessive.

### 9.3. Mute/Volume Controls

M16.5 may add simple UI-local controls:

* mute SFX;
* mute ambience;
* SFX volume;
* ambience volume.

But save/load persistence for these settings is out of scope.

If controls are added, their values are session-local only.

## 10. Scene Ownership

### 10.1. `GameRoot`

`GameRoot` may own or instantiate the Audio Controller.

Allowed responsibilities:

* connect facade signals to audio events;
* route screen-level audio events;
* start/stop ambience on game start/reset;
* play Game Over sound.

`GameRoot` must not:

* compute gameplay results;
* mutate gameplay state;
* call owner modules directly.

### 10.2. `GameScreen`

`GameScreen` may emit UI-local audio events for:

* phase panel shown;
* action failed shown;
* core panel action result received.

`GameScreen` must not compute outcome-specific sounds by gameplay formulas.

### 10.3. `CardView`

`CardView` may emit or request:

* hover sound;
* selected sound;
* click sound.

CardView must not:

* compute card availability;
* compute price;
* call `GameStateManager.state`;
* call owner modules.

### 10.4. `MarketPanel`

`MarketPanel` may play/request:

* card select;
* purchase success;
* denied purchase;
* end market click.

Purchase success sound must require facade success.

### 10.5. `ActionPanel`

`ActionPanel` may play/request:

* card select;
* target select;
* attack execute;
* attack blocked;
* attack success;
* discard;
* denied action.

Attack/block/success sounds must use facade result or view/log information only.

### 10.6. `StreetDealPanel`

`StreetDealPanel` may play/request:

* option hover/select;
* successful Street Deal choice;
* denied choice.

It must use canonical option IDs:

```text
option_a
option_b
```

### 10.7. `ContactPanel`

`ContactPanel` may play/request:

* contact select;
* contact unlock success;
* denied contact selection.

### 10.8. `ContractPanel`

`ContractPanel` may play/request:

* claim success;
* denied claim.

### 10.9. `GameOverScreen`

`GameOverScreen` may play/request:

* game over reveal sound;
* new game click.

It must not recompute winner.

## 12. Tactility Rules

### 12.1. Audio + Feedback Alignment

Audio should align with M16.4 visual feedback.

Examples:

* hover lift pairs with soft card sound;
* selected outline pairs with card tap;
* denied shake pairs with dull knock;
* purchase pulse pairs with chip/card placement;
* attack flash pairs with impact;
* block badge pulse pairs with shield/metal/card slap;
* phase transition pairs with soft table shift.

### 12.2. No Audio Spam Rule

Audio must avoid spam.

Required safeguards:

* hover sound cooldown;
* no repeated denied sound every frame;
* no phase sound loop;
* no log sound for every tiny refresh unless guarded;
* no ambience restart on every state_changed.

### 12.3. Short Sound Rule

SFX should be short.

Recommended:

* hover: very short;
* click/select: short;
* denied: short and clear;
* purchase/attack/block: short;
* phase transition: short;
* game over: can be longer but still controlled.

## 15. Defect Fix Policy

M16.5 may fix only:

* confirmed audio/tactility defect;
* scene parse/runtime error;
* duplicate sound spam issue;
* missing UI-local audio state reset;
* signal connection bug;
* audio static boundary gap;
* missing audio event mapping needed by this milestone;
* test-confirmed regression.

For each fix:

1. show reproduction or failing test;
2. identify owner file/module;
3. add regression test;
4. make minimal change;
5. run targeted and affected tests.

M16.5 must not fix gameplay by changing audio.
