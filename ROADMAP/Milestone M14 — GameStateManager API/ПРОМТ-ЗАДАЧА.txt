ARCHITECTURE CONTRACT:

- `GameStateManager.gd` extends `Node`.
- Autoload name: `GameStateManager`.
- Он владеет `var state: Dictionary = {}`.
- Это единственный UI-facing gameplay facade и active state owner.
- Facade не содержит formulas, effects, scoring или random logic.
- Facade не зависит от UI scenes/Control nodes.
- Resources не мутируются; второй runtime store не создаётся.
- `GameStateManager.gd` строго меньше 250 строк.
- Каждый другой `.gd` файл также меньше 250 строк.

Если facade приближается к лимиту:

- вынеси view shaping в `GameViewBuilder.gd`;
- используй existing owner selectors;
- оставь public methods короткими;
- не прячь gameplay rules в generic helper.

ACTIVE STATE И COMMIT:

- До успешного setup active state пуст.
- После setup state обязан проходить GameStateValidator.
- Наружу нельзя передавать writable reference на active state.
- `get_state_snapshot()` возвращает deep copy.
- Logic modules получают только `state.duplicate(true)`.
- Failure оставляет active state полностью неизменным.
- Success заменяет active state validated returned state.
- Failed replacement setup сохраняет предыдущую active game.

Каждый public mutator:

1. Проверяет active game и required payload shape, если применимо.
2. Создаёт deep working copy.
3. Делегирует owner module.
4. Не commit-ит result при `ok != true`.
5. Проверяет returned state и валидирует его.
6. Commit-ит только полностью успешный validated result.
7. Эмитит только подходящие signals.
8. Возвращает structured result без live state reference.

Facade проверяет routing/shape; gameplay legality принадлежит owner module.
Не дублируй owner validation и rule resolution.

RESULT И ERROR CONTRACT:

Сохрани fields `ok`, `error`, `state`, применимый `log_entries`
и module-specific fields owner result.

- Success использует `ValidationErrors.OK`.
- Failure использует stable documented constant.
- Ad-hoc UI-facing strings запрещены.
- Отсутствующий error добавляй только при документированном meaning.
- Если meaning не определён, создай `OQ-*`.

SIGNALS:

- `state_changed(state: Dictionary)`
- `action_failed(error: String, result: Dictionary)`
- `phase_changed(phase_id: String)`
- `game_started(state: Dictionary)`
- `game_ended(result: Dictionary)`

Правила:

- `state_changed` только после successful commit;
- failure не эмитит `state_changed`;
- `action_failed` допустим после public failure;
- `phase_changed` только при реальном изменении phase;
- game start/end signals только после соответствующего commit;
- selectors не эмитят signals;
- Dictionary payloads являются deep copies/safe results.

DOMAIN BOUNDARIES:

- Setup делегирует factory, contracts, roles, AI, Turf и phase owners.
- Phase fields меняет только GamePhaseController.
- Purchase и rebuild делегируются Economy/Market owners.
- Rebuild является dedicated API, не fake card ID.
- Combat validation/preview/resolve/discard принадлежат CombatEngine.
- Contract claim/reward принадлежат ContractLogic.
- Street Deal effects и debts принадлежат соответствующим owners.
- Contact selection/activation принадлежит ContactLogic.
- AI calls делегируются AIBotController и не bypass-ят Market/Combat.
- Turf Level не имеет runtime setter в MVP.

Facade не должен вручную:

- менять Nal, VP, hands, buildings или defenses;
- двигать cards;
- применять rewards/effects;
- выставлять phase/readiness/action flags;
- вычислять prices, combat outcome, AI scores или Turf modifiers.

SETUP CONTRACT:

`start_new_game(config)` принимает:

- non-empty String `game_seed`;
- Integer `turf_level` в диапазоне 0..10;
- valid `selected_role_id`;
- valid offered `selected_contract_id`.

Он использует existing M0–M13 setup pipeline, сохраняет random call order,
валидирует final state и commit-ит только полный success.

Setup offer preview:

- не мутирует active state;
- использует временный deterministic state из config;
- возвращает explicit draft/result;
- не создаёт hidden global draft store;
- одинаковый config даёт одинаковый result;
- `start_new_game` повторно валидирует selection.

Если repository уже имеет другой явно документированный safe setup contract,
не создавай второй competing pipeline; укажи расхождение в audit.

SELECTOR SAFETY:

Каждый selector/preview:

- не мутирует state или nested data;
- не меняет random step/history/tag;
- не пишет logs;
- не consume-ит modifiers/flags;
- не продвигает phase;
- не эмитит signals;
- возвращает safe view/deep copy или documented primitive;
- безопасно обрабатывает no active game и invalid IDs.

`GameViewBuilder.gd`, если создан:

- только собирает read-only UI view;
- не владеет gameplay rules;
- не вызывает random и не пишет logs;
- не возвращает live references;
- остаётся меньше 250 строк.
