Проект: The Turf / Передел
О чём продукт

“The Turf” / “Передел” — desktop single-player card/board mini-game внутри визуальной новеллы.

Разработка строго по milestone-порядку:

Сейчас M6: получение/расход Nal, deterministic Income, price calculation, shared market, purchase validation/resolution, Cops upkeep, District Control rebuild.

Источник истины: res://docs/prd/.
Не менять цены, market pools, card placement, modifier order.
При конфликте использовать resolved FIX-*; unresolved ambiguity оформлять как OQ-*.

Стек

Архитектура

Dependency direction:

UI → GameStateManager → logic modules → catalogs/resources/constants

M6 owner files:

PriceLogic.gd — read-only price/protection calculations.
MarketLogic.gd — market generation и economy orchestration.
IncomeLogic.gd — deterministic income и Cops upkeep.
PurchaseValidator.gd — read-only purchase/rebuild validation.
PurchaseResolver.gd — successful state mutation and placement.

Invariants:

Nal/VP never negative.
Price preview / validation не мутируют state/random.
Failed purchase/rebuild не мутирует state и не consumes flags/modifiers.
Market generation использует shared state["random"].
Income roll использует roll_d6_pair: ровно 2 random steps на player.
Purchase работает на deep copy; commit только при success.
Resource definitions immutable.
Gameplay UI отсутствует.
Каждый .gd файл < 250 строк.

Договорённости по коду
Используй card catalogs/definitions; не парси effect_summary.
Stable IDs/errors брать из M1.
Modifier order: base/scaled → rebuild override → role hook → Turf hook → temporary/contact modifiers → clamp.
Temporary/contact flags consume только после successful matching purchase.
Starting resources в M6: base values + explicit integration points; Role/Turf setup effects принадлежат M8/M12.
Debt, Contract и AI behavior не fake-stub.
Ориентиры длины prompt-файлов примерные, не acceptance criteria.
Не ослабляй tests и не скрывай failed/not-run checks.

Не реализуй combat, RoleLogic, ContractLogic, DebtLogic, ContactLogic, TurfLevelLogic, AI purchasing, GameStateManager или UI.
