# Проект: The Turf / Передел

## О чём продукт

The Turf / «Передел» — desktop single-player card/board mini-game
внутри визуальной новеллы.

В матче участвуют один человек и три локальных AI.
Партия длится ровно 15 раундов и завершается Game Over.

Текущий этап: Milestone M15 — Integration and Replay.

## Стек

- Godot 4.6.2 stable.
- GDScript со статической типизацией, где это практично.
- GUT.
- `.tres` Resources для static definitions.
- Dictionary snapshots для runtime state.
- GameStateManager как Autoload facade.
- Windows/Linux first.
- Без web stack.

## Архитектура

- Logic-first, data-driven.
- Gameplay rules принадлежат owner modules M5–M13.
- GameStateManager M14 — единственный public gameplay facade.
- Integration tests проверяют реальные связи между modules.
- Replay tests повторяют одинаковый seed и одинаковые commands.
- Один shared `state["random"]` определяет весь gameplay random.
- Replay runner является test fixture, а не gameplay system.
- Runner вызывает GameStateManager и не мутирует state напрямую.
- Replay не является save/load или persistence.
- Tests используют snapshots из GameStateManager/factory.
- Resources остаются immutable.
- Failed commands не изменяют active state.
- Selectors не consume random.
- WinnerResolver не использует random.
- UI не требуется для выполнения M15 flows.

## Договорённости по коду

- Каждый `.gd` файл меньше 250 строк.
- Tests не содержат test-only gameplay rules.
- Replay script хранит explicit commands и payloads.
- Human choices выполняются через GameStateManager.
- AI выполняется через M14 AI API/phase orchestration.
- Runner не пишет напрямую в `GameStateManager.state`.
- Runner не вызывает low-level mutators в обход facade.
- Runner не использует forbidden random.
- Same seed + same script + same code дают identical result.
- Final `random.step` является обязательной частью comparison.
- Gameplay arrays не сортируются и не нормализуются произвольно.
- Normalize удаляет только explicitly debug-only fields.
- Market history собирается в test trace, а не в новый state field.
- Replay comparison выполняется in-memory; persistence не требуется.
- Bugs сначала воспроизводятся тестом, затем исправляются owner module.
- Gameplay rules и deterministic call order не меняются ради pass.
- Все resolved FIX-001–FIX-030 обязательны.
- Нерешённое поведение фиксируется как `OQ-*`.
- Ориентиры длины промтов приблизительные.

## Что НЕ входит в текущий milestone

- Новые gameplay mechanics, cards, roles, deals или AI profiles.
- Изменение balance, formulas, IDs, rewards или probabilities.
- Новый random algorithm или отдельный random stream.
- Прямое редактирование state тестовым runner.
- Gameplay save/load, campaign persistence или replay file format.
- Final UI, screens, widgets и animations из M16.
- Production polish, analytics и hardening из M17.
- Подмена integration flow mocks вместо real modules.
- React, TypeScript, Tailwind, Zustand, Docker, backend и routing.
