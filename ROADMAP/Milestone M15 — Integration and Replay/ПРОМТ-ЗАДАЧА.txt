REPLAY RUNNER CONTRACT:

Обязательный helper:

```text
res://tests/fixtures/ReplayScriptRunner.gd
```

Минимальный API:

- `run_scripted_game(seed: String, script: Array[Dictionary])`
- `normalize_snapshot(snapshot: Dictionary)`

Разрешено добавить небольшие typed helpers для command dispatch,
trace capture и result validation, если каждый file меньше 250 строк.

Runner должен:

1. Сбрасывать GameStateManager перед run.
2. Создавать game только через `start_new_game`.
3. Выполнять human commands через public M14 API.
4. Выполнять AI через public/phase-safe M14 API.
5. Продвигать phases через `advance_phase` и dedicated phase methods.
6. Проверять `result["ok"]` после каждого required command.
7. Проверять GameStateValidator в значимых checkpoints.
8. Собирать deterministic trace без mutation active state.
9. Возвращать final deep snapshot и execution metadata.
10. Завершаться structured failure с command index при invalid script.

Runner запрещено:

- писать в `GameStateManager.state`;
- заменять production state fixture snapshot во время end-to-end run;
- вызывать owner mutators в обход facade;
- вызывать SeededRandom/SeededPicker самостоятельно;
- вычислять gameplay choices случайно;
- изменять command payload, чтобы failed step внезапно прошёл;
- игнорировать failed result и продолжать run;
- использовать wall-clock time, UUID или OS random;
- сохранять/загружать gameplay replay с диска.

SCRIPT CONTRACT:

Script — `Array[Dictionary]` explicit commands в stable order.

Каждый command должен иметь понятный operation ID и explicit payload.
Operation ID отображается только на существующий GameStateManager API.

Допустимые группы commands:

- setup;
- phase advancement;
- human Market buy/end;
- human Action attack/discard/end/skip;
- Street Deal selection;
- Contact selection/activation;
- Contract claim;
- AI Market/Action orchestration.

Не создавай «умные» self-healing commands вроде:

- buy_any_valid;
- silently_skip_on_failure;
- choose_whatever_is_available;
- mutate_state_until_valid.

Для fixed seed используй explicit valid setup selection и explicit human
choices. Если deterministic outputs нужны для payload, сначала исследуй
run, затем зафиксируй seed/script как immutable test fixture.

Replay script не должен зависеть от iteration order неупорядоченного
Dictionary. Stable IDs и Arrays используются в documented order.

RUN RESULT:

Используй stable test result shape, содержащий минимум:

- `ok`;
- `error`;
- `failed_command_index`;
- `failed_command`;
- `state`;
- `random_step`;
- `trace`;
- `commands_executed`.

Это test-helper result, а не новый gameplay API.
Не добавляй его в GameState schema.

TRACE CONTRACT:

Trace нужен для диагностики и deterministic comparisons.

Записывай read-only checkpoints:

- command index/operation;
- round;
- phase;
- random step;
- selected high-level result IDs;
- market snapshot при каждом входе в Market;
- Street Deal ID/option при resolution;
- contact offer IDs, если offer появился;
- AI profile assignment и strong AI из setup;
- winner/game result в конце.

Trace не должен:

- содержать Node/Resource references;
- содержать wall-clock timestamps;
- влиять на gameplay;
- добавляться в active GameState;
- заменять final snapshot comparison.

`market history` из Test Plan означает test trace последовательности
market snapshots. Не добавляй недокументированное
`state["market_history"]`.

SNAPSHOT NORMALIZATION:

Replay A и B сравниваются по normalized deep snapshots.

`normalize_snapshot()` может исключать только:

- wall-clock timestamps в debug-only logs, если они реально существуют;
- editor-only metadata;
- optional `random.history`, если конкретный test не проверяет history.

Нельзя игнорировать:

- Nal и VP;
- market и hands;
- engine/defense/status;
- contracts, contacts и debts;
- phases и round;
- winner и game_result;
- combat/gameplay logs;
- random seed/step/last tag;
- gameplay arrays и их order.

Не сортируй gameplay arrays для сокрытия order mismatch.
Не округляй gameplay values.
Не удаляй failed/used/consumed flags.

Предпочитай gameplay logs без timestamps.
Если debug-only normalization не нужна, сравни snapshots напрямую.

FULL-ROUND SMOKE FLOW:

`test_full_round_flow.gd` должен выполнить минимум:

1. Valid deterministic setup.
2. Setup -> Income.
3. Resolve Income для всех четырёх players.
4. Income -> Market с generated market.
5. Human выполняет explicit valid Market behavior или end.
6. Все AI завершают Market через AI facade.
7. Market ждёт readiness всех четырёх players.
8. Market -> Action с order `player_1, ai_1, ai_2, ai_3`.
9. Human выполняет explicit Action behavior или end.
10. AI выполняют Actions через AI facade.
11. Action завершается только после всех `action_done`.
12. Round 1 -> Income round 2.
13. State валиден на каждом phase boundary.

Проверь:

- no direct state mutation;
- phase/round invariants;
- active action player order;
- Market/Action flags reset;
- expected random step monotonic growth;
- no Game Over после round 1;
- failed early transition не мутирует state.

STREET DEAL INTEGRATION:

Проверь отдельным integration flow:

- Action round 4 -> Street Deal;
- аналогичные phase rules для rounds 8 и 12;
- human выбирает explicit valid option;
- AI не делает Street Deal choice;
- explicit AI side effects остаются допустимыми;
- contact offer/selection проходит через facade, если сценарий его создаёт;
- Street Deal completion -> Income next round;
- used deal IDs и debts/temporary effects сохраняются корректно.

Не форсируй Street Deal прямым изменением phase в end-to-end replay.
Targeted module fixtures допустимы только в focused integration tests.

FULL 15-ROUND SCRIPTED FLOW:

Создай fixed scenario, который:

1. Использует stable seed, например `test_seed_replay`.
2. Использует valid deterministic role/contract/Turf setup.
3. Проходит rounds 1..15 через GameStateManager.
4. Разрешает Income каждого round.
5. Завершает human и AI Market participation.
6. Завершает human и AI Action participation.
7. Разрешает Street Deals после rounds 4, 8, 12.
8. Обрабатывает pending contact selection, если она возникает.
9. Выполняет explicit contract claim, если script это предусматривает.
10. Не создаёт round 16.
11. Заканчивается `current_phase == game_over`.
12. Имеет valid `winner_id` и populated `game_result`.

Human script может быть минимальным, но не должен обходить phases.
AI decisions должны выполняться настоящей M13 logic.

Scenario должен иметь loop/command guards:

- max 15 rounds;
- finite command count;
- diagnostic failure при stalled phase;
- diagnostic failure при repeated unchanged checkpoint;
- no silent infinite loop.

Не меняй seed после того, как scenario и expectations зафиксированы.

REPLAY DETERMINISM TEST:

`test_replay_determinism.gd`:

1. Выполняет scenario A с fixed seed/script.
2. Сохраняет result и deep final snapshot in-memory.
3. Полностью reset-ит GameStateManager/test state.
4. Выполняет scenario B с тем же seed/script.
5. Убеждается, что оба runs successful.
6. Сравнивает normalized final snapshots exactly.
7. Сравнивает deterministic traces.

Отдельно assert:

- same winner ID и game result;
- same final scores;
- same market trace;
- same AI profiles;
- same strong AI;
- same Street Deal sequence/used IDs;
- same contact offer trace/state;
- same contract state;
- same debts;
- same combat log;
- same commands executed.

Не считай test достаточным, если оба runs завершились одинаковой ошибкой.

RANDOM STEP CONSISTENCY TEST:

`test_replay_random_step_consistency.gd` проверяет:

- starting random seed одинаков;
- step никогда не уменьшается в run;
- selectors/checkpoint capture не меняют step;
- final step A == final step B;
- final step совпадает со snapshot;
- trace random steps совпадают command-by-command;
- no secondary/local random stream;
- same command не расходует разное число steps между runs.

Не хардкодь произвольный final step до первого validated run.
После фиксации scenario допустимо lock-нуть expected final step
как compatibility assertion, если repository использует такой pattern.

INTEGRATION COVERAGE:

Required integration tests должны доказывать:

- setup связывает factory, role, contract, AI и Turf modifiers;
- Income связывает dice, upkeep, debts, contacts и contracts;
- Market связывает generation, pricing, purchases и hooks;
- Combat связывает War resolution, defenses, contracts и contacts;
- Street Deals связывают options, debts и contact offers;
- AI использует MarketLogic/CombatEngine и phase boundaries;
- GameStateManager commit safety сохраняется cross-module;
- Game Over связывает phase controller, winner и Turf Level 10 rule.

Не дублируй в integration tests все unit permutations.
Проверяй module boundaries, meaningful side effects и invariants.

NON-VACUOUS COVERAGE:

Перед финальным gate составь coverage report по replay scenario:

- какие random consumers реально сработали;
- какие phases пройдены;
- какие hooks/events возникли;
- какие поля trace остались пустыми.

Пустой trace допустим, если event не возник по script, но нельзя заявлять,
что replay доказал behavior этого event. Сохрани соответствующий unit или
focused integration coverage вместо искусственного изменения gameplay.

BUG FIX POLICY:

Если replay расходится:

1. Найди первый divergent command/checkpoint.
2. Сравни phase, state и random step до divergence.
3. Определи owner module.
4. Проверь PRD и resolved FIX.
5. Добавь минимальный regression test.
6. Исправь production code, а не normalize/expected result.
7. Перезапусти owner unit tests и оба replay tests.

Запрещено «исправлять» mismatch через:

- удаление gameplay fields из snapshot;
- сортировку meaningful arrays;
- reset/rewind random step;
- дополнительные random calls для выравнивания;
- test-only branches в production;
- ослабление assertion;
- direct state patch в runner.
