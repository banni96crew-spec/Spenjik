ERROR TEXT HARDENING:
Построй coverage table:
- error code;
- owner/return site;
- current UI text;
- fallback;
- test.
Правила:
- используй actual constants из `ValidationErrors.gd`;
- не меняй semantics stable codes;
- `OK` не показывай как failure;
- unknown non-empty code получает generic readable fallback;
- debug context сохраняет исходный code;
- empty/malformed value обрабатывается безопасно;
- не добавляй error constants только ради wording;
- не дублируй gameplay validation в ErrorTextMap;
- не меняй failed result на success.
Проверь errors setup, phase, market, combat, contracts,
Street Deals/Debts, contacts и facade validation.
Если semantic contract не определён, создай `OQ-*`, не угадывай.

LOG READABILITY HARDENING:
Сохрани authoritative fields:
- `id`, `round`, `phase`, `event_type`;
- `actor_id`, `target_id`, `card_id`;
- `summary`, `details`.
Правила:
- log остаётся append-only;
- порядок, IDs и event types сохраняются;
- UI не создаёт gameplay entries;
- formatting делается в UI/helper layer, где возможно;
- state хранит IDs, display text не подменяет их;
- empty/optional/long entries отображаются безопасно;
- expanded details — UI-local presentation state;
- не добавляй wall-clock timestamps;
- cosmetic changes не должны менять replay outcome.
Проаудируй required events:
- match/round/phase;
- income/market/action;
- purchases/rebuild;
- attacks/blocks/discards;
- contracts/Street Deals/debts/contacts;
- AI activity;
- Game Over/winner.
Если event отсутствует вопреки owner PRD:
1. Докажи defect test/reproduction.
2. Минимально исправь owning log builder.
3. Не меняй gameplay effects.
4. Добавь regression test.
5. Перезапусти replay.

DEBUG SNAPSHOT:
Required path:
- `user://debug_run_snapshot.json`
Обязательные свойства:
- сериализуется detached Dictionary snapshot;
- source и active state не мутируются;
- random step, phase и combat log не меняются;
- snapshot не required для normal gameplay;
- gameplay не читает snapshot;
- write failure не ломает матч;
- facade не commit-ит state и не emit-ит gameplay signals;
- debug summary read-only и deterministic;
- JSON не содержит Node/Resource runtime references.
Запрещено:
- SaveManager, save slots, autosave, continue/load flow;
- gameplay restore или чтение snapshot при старте;
- migration system;
- gameplay dependency на DebugSnapshotManager.
`load_debug_snapshot()` optional и не подключается к MVP gameplay.
Tests:
- valid JSON создан и содержит expected state fields;
- source snapshot/state/random/logs unchanged;
- failure path safe;
- normal run работает без snapshot;
- debug summary одинаков для одинакового state.
Filesystem test удаляет только созданный test file
и не зависит от machine-specific absolute path.
