The Turf / «Передел» — desktop single-player card/board mini-game с одним human player и тремя local AI.

M0–M16.2 считаются prerequisites, но readiness нужно проверить по repository/tests.

```text
UI -> GameStateManager -> logic modules -> catalogs/resources/constants
```

M16.3 не меняет deterministic replay.

## Atmosphere goals

UI должен ощущаться как:

* тёмный игровой стол;
* криминальная настольная игра;
* грязная бумага/сукно/матовый картон;
* подпольное казино/клуб;
* старые карты на столе;
* физический tabletop, а не spreadsheet/debug panel.

UI не должен ощущаться как:

* default Godot;
* clean prototype;
* mobile UI;
* fantasy card game;
* neon cyberpunk;
* web dashboard;
* debug tool.

---

## Visual style requirements

Palette:

* black / near-black;
* dirty ivory / warm off-white text;
* muted gray;
* brown grime;
* dark desaturated red accents;
* no bright saturated palette.

Theme:

* dark panels;
* worn borders;
* subtle separators;
* readable labels;
* muted buttons;
* consistent disabled state;
* consistent hover/selected feedback;
* no default blue/gray Godot style.

Background:

* tabletop surface;
* subtle noise/grime;
* not too bright;
* not distracting;
* no realistic photo dependency unless asset is project-local and safe;
* no external absolute paths in production files.

Panels:

* should feel like placed objects/cards/papers on table;
* should not look like plain debug rectangles;
* borders and backgrounds consistent;
* side panels secondary, not dominant.

Text:

* readable at 1280×720;
* hierarchy clear;
* off-white on dark;
* warnings/errors can use controlled red;
* no tiny unreadable decorative font.

## Theme requirements

Theme should define or consistently apply:

* default font color;
* disabled font color;
* warning/error accent;
* panel background;
* panel border;
* button normal/hover/pressed/disabled;
* label readability;
* scrollbar visibility if used;
* card/table-compatible colors.

If a global Theme already exists:

* update it safely;
* do not create parallel conflicting themes.

If no clean Theme exists:

* create one project-local Theme resource;
* apply through Main/GameRoot;
* keep theme presentation-only.

---

## Background requirements

Tabletop background should:

* be dark;
* not distract from cards/buttons;
* support 1280×720 and 1920×1080;
* scale safely;
* not require external absolute image paths;
* not import malformed reference files;
* not be used as gameplay state.

Allowed approaches:

* ColorRect + subtle layered panels;
* TextureRect with project-local safe texture;
* procedural StyleBox-based background;
* simple grime/noise asset if added safely.

Avoid:

* giant uncompressed image assets without reason;
* external absolute paths;
* runtime loading from `C:\...`;
* full-screen image that makes text unreadable.

## Defect fix policy

Можно исправлять только:

* atmosphere/theme defect;
* scene parse/runtime error;
* unreadable UI due to style;
* unsafe external path;
* UI static boundary gap;
* test-confirmed regression.

Для каждого fix:

1. Покажи reproduction/failing test.
2. Укажи owner file/module.
3. Добавь regression test.
4. Внеси минимальное изменение.
5. Запусти targeted, affected и full suites.

Не исправляй gameplay через atmosphere/theme.
Не подгоняй expected result теста под bug.
