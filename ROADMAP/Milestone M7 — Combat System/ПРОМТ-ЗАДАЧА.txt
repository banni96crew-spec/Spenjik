ATTACK CONTRACT:

Нормализованный payload:

```gdscript
{
    "attacker_id": ...,
    "target_id": ...,
    "card_id": ...,
    "mode": "",
    "modifiers": [],
    "engine_target_card_id": ""
}
```

Primary cards:

- Thug;
- Bruiser;
- Cleaner;
- Saboteur;
- Federal Raid.

Insider является только modifier для Thug.
Он не может быть primary card и должен находиться в hand.

Порядок validation:

1. Action phase.
2. Required payload fields.
3. Attacker существует.
4. Target существует.
5. Attacker не равен target.
6. Attacker равен `active_action_player_id`.
7. Primary card допустима.
8. Primary card находится в hand.
9. Modifiers допустимы и находятся в hand.
10. Required mode указан.
11. Mode допустим для карты.
12. Target удовлетворяет effect requirements.
13. Saboteur engine target явно выбран и допустим.

Используй существующие error IDs, включая эквиваленты:

- `INVALID_PHASE`;
- `INVALID_TARGET`;
- `INVALID_ACTION_CARD`;
- `ATTACK_MODE_REQUIRED`;
- `INVALID_ATTACK_MODE`.

Не создавай ad hoc ID.

MUTATION CONTRACT:

- Validation, preview и selectors не мутируют state.
- Failed validation не расходует карты и не вызывает hooks.
- Failed validation не меняет `last_attacked_by`.
- Resolution работает с `state.duplicate(true)`.
- Любая валидная resolved атака расходует primary card.
- Blocked valid attack также расходует primary card.
- Валидный Thug + Insider расходует обе карты.
- Удаляется ровно один экземпляр каждой карты.
- Successful и blocked attack обновляют `last_attacked_by`.
- Combat не завершает action и не меняет phase.
- Combat не использует и не изменяет RNG.
- Blocked destroy не переключается на steal.

CARD RULES:

Thug:

- mode не нужен;
- steal до 6 Nal с учётом protected Nal;
- active Cops блокирует Thug без Insider;
- Cops остаётся active и не расходуется;
- valid Insider обходит Cops.

Bruiser:

- обязательный mode: `steal_nal` или `destroy_stash`;
- steal до 8 Nal с учётом protected Nal;
- destroy требует Stash у target;
- active Cartel блокирует destroy и остаётся active;
- успешный destroy: Stash -1, target VP -1, attacker Nal +3.

Cleaner:

- обязательный mode: `steal_nal` или `destroy_workshop`;
- steal до 14 Nal с учётом protected Nal;
- destroy требует Workshop у target;
- active Cartel блокирует destroy и становится depleted;
- успешный destroy: Workshop -1, target VP -2,
  attacker Nal +5, `skip_next_action = true`.

Saboteur:

- mode не нужен;
- attacker явно выбирает `engine_target_card_id`;
- допустимы Informant, Laundry, Accountant и Brothel;
- target обязан владеть выбранным engine;
- случайный выбор запрещён;
- active Judge блокирует атаку и становится inactive/none;
- success уничтожает только выбранный engine.

Federal Raid:

- обязательный mode: `destroy_district`;
- требует `district_control > 0`;
- Cops, Cartel и Judge его не блокируют;
- success: district control -1, target VP -3,
  `can_rebuild_district_for_8 = true`.

VP и Nal clamp к 0.
Federal Raid не реализует rebuild API заново.

PROTECTED NAL:

Используй единый M6 API `PriceLogic.get_protected_nal(...)`
или его фактический эквивалент.

```text
stealable = min(card_max, max(0, target_nal - protected_nal))
```

Steal 0 остаётся успешной атакой и расходует карту.
Не дублируй protected Nal formula.

DEFENSE И PREVIEW:

Проверяй максимум одну релевантную защиту:

- Cops против Thug без Insider;
- Cartel против destroy Stash/Workshop;
- Judge против Saboteur.

Defense result содержит как минимум:

- `blocked`;
- blocker;
- side effects;
- description.

Preview должен показывать применимые данные:

- validation/error;
- would be blocked и blocker;
- protected/max/stealable Nal;
- VP loss и Nal gain;
- уничтожаемый объект;
- skip, Cartel и Judge side effects;
- расходуемые карты.

Preview сообщает эффекты, но ничего не применяет,
не пишет log в state и не вызывает hooks.

Selectors возвращают только фактически допустимые targets/engines
и также не имеют side effects.

DISCARD:

`discard_war_card`:

- принимает только War-карту из hand игрока;
- удаляет ровно один экземпляр;
- создаёт discard log;
- не применяет combat effect;
- не меняет `last_attacked_by`, phase или action;
- не вызывает success hooks;
- не использует RNG.

RESULT И HOOK BOUNDARY:

Combat result должен быть структурированным и содержать эквиваленты:

- ok/error;
- attacker, target, card, mode и modifiers;
- blocked, blocker и success;
- effect result;
- cards consumed;
- contract/contact result slots;
- log entries;
- resulting state.

Не создавай fake `ContractLogic` или `ContactLogic`.
Не реализуй contract progress, contact offers или AI behavior.

Result/hook event должен позволять будущим модулям определить:

- successful unblocked attack;
- уничтожение Stash, Workshop или district control;
- уничтожение engine card;
- attacker, target и владельца объекта;
- blocked outcome.

Используй существующий hook convention.
Если его нельзя определить без выдумывания rules/API,
создай `OQ-*` и остановись на этом integration point.
