# ПРОМТ-КОНТЕКСТ — M16.4 Animation & Feedback

## Проект: The Turf / Передел

### О чём продукт

The Turf / «Передел» — desktop single-player card/board mini-game внутри визуальной новеллы.

В матче участвуют:

* 1 human player;
* 3 local AI;
* партия длится 15 раундов;
* финал — Game Over.

Текущий этап: **Milestone M16.4 — Animation & Feedback**.

Это дополнительный UI-polish milestone после M16.1/M16.2/M16.3 и до M16.5/M17.

Цель M16.4 — добавить визуальный feedback и лёгкие UI-анимации, чтобы игра ощущалась живее:

* hover/selected feedback;
* feedback покупки;
* feedback атаки;
* feedback block/defense;
* feedback denied action;
* phase transition;
* card movement/reveal ощущение;
* Game Over reveal.

M16.4 **не меняет gameplay rules, balance, deterministic replay, public facade contracts или state schema**.

## Архитектура

Dependency direction:

```text
UI -> GameStateManager -> logic -> catalogs/resources/constants
```

Правила:

* UI является view/input layer.
* Runtime state принадлежит GameStateManager.
* UI вызывает gameplay только через GameStateManager.
* UI не читает и не мутирует `GameStateManager.state`.
* Animation/feedback layer является presentation-only.
* Animation не меняет gameplay state.
* Animation не consume gameplay random.
* Animation не меняет replay outcome.
* Animation не должна быть dependency для logic modules.
* Gameplay modules не зависят от UI scenes/assets/animation.
* UI может иметь локальный busy/presentation lock, но не gameplay lock.
* M15 replay determinism и M16/M16.1/M16.2/M16.3 UI boundaries должны остаться зелёными.

## Visual direction

Пользователь хочет UI в сторону тёмного tabletop card-game presentation.

Опорное направление:

* Inscryption-like physical card table mood, но без копирования чужого IP;
* dirty criminal board-game table;
* dark matte cards;
* worn print feel;
* controlled feedback, не arcade overload;
* тяжёлые, короткие, тактильные движения;
* no flashy mobile juice;
* no neon VFX spam;
* no fantasy magic particles.

## Договорённости по коду

* M16.4 не меняет gameplay rules.
* Не менять card effects, prices, IDs, formulas, phase order или AI policy.
* Не менять deterministic random.
* Не менять replay scripts.
* Не менять state schema.
* Не менять public GameStateManager contracts без подтверждённого blocker.
* Не добавлять save/load, persistence, campaign или profiles.
* Не читать raw `GameStateManager.state`.
* Не вызывать low-level owner modules из UI.
* Animation может слушать facade/UI signals, но не owner modules.
* Animation не должна быть обязательной для gameplay execution.
* Animation failure не должна ломать матч.
* UI feedback не должен коммитить gameplay state.
* UI feedback может блокировать повторный клик на время короткого presentation lock, но не должен менять phase flow.
* Каждый `.gd` файл строго меньше 250 строк.
* Static scans нельзя ослаблять.
* UI boundary tests должны оставаться зелёными.
