# Проект: The Turf / Передел

## О чём продукт

“The Turf” / “Передел” — desktop single-player card/board mini-game внутри визуальной новеллы.

Разработка идёт строго по milestone-порядку:

M0 Project Bootstrap
→ M1 Constants and IDs
→ M2 Resource Schemas and Catalogs
→ M3 Deterministic Random
→ M4 Game State Schema
→ M5 Core Loop and Phases
→ последующие gameplay-модули

Матч длится ровно 15 раундов. Phase system управляет тем, когда запускаются игровые системы, но не определяет, как economy, combat, AI, contracts, contacts или Street Deals работают внутри.

Документы в `res://docs/prd/` — источник истины. Не придумывай readiness conditions, side effects или tie-break rules. При конфликте проверь resolved `FIX-*`; нерешённую неоднозначность оформи как `OQ-*`.

## Стек

- Godot 4.6.2 stable.
- GDScript и GUT.
- Dictionary snapshots для runtime state.
- `.tres` Resources для static data.
- Windows и Linux как первые export targets.
- Без web stack.

## Архитектура

Будущее направление зависимостей:

UI → GameStateManager → logic modules → catalogs/resources/constants

M5 вводит orchestration layer:

- `GamePhaseController.gd` владеет round/phase transitions.
- `WinnerResolver.gd` вычисляет deterministic final result.
- `GameStateValidator.gd` проверяет state до и после transition.
- UI не меняет round, phase, action order или flags напрямую.

Core loop:

Setup → Income → Market → Action

После Action:

- rounds 4, 8, 12 → Street Deal;
- rounds 1–3, 5–7, 9–11, 13–14 → следующий Income;
- round 15 → Game Over.

Критические invariants:

- Разрешены только `PhaseIds.ALL`.
- Round всегда находится в диапазоне 1–15.
- Round 16 никогда не создаётся.
- Action order фиксирован: `player_1`, `ai_1`, `ai_2`, `ai_3`.
- Outside Action `active_action_player_id == ""`.
- Invalid transition и failed validation не мутируют original state.
- WinnerResolver не использует random и не изменяет random step.
- Phase controller оркестрирует модули, но не реализует их gameplay rules.
- Каждый `.gd` файл содержит менее 250 строк.

## Договорённости по коду

- Phase operations работают с duplicated working state.
- Commit/returned state появляется только после successful validation.
- Controller не рассчитывает income, prices, attacks, AI choices или Street Deal effects.
- До появления owner modules используй только явно разрешённые integration hooks; не создавай fake gameplay implementations.
- Winner result соответствует M4 schema.
- Tie-break calculations не добавляют VP и не мутируют players.
- Используй constants, а не repeated phase/player strings.
- Ориентиры длины prompt-файлов примерные и не являются acceptance criteria.
- Не ослабляй tests и не скрывай failed/not-run checks.

## Что НЕ входит в текущий milestone

Не реализуй IncomeLogic, MarketLogic, purchase flow, CombatEngine, AI decisions, role/contract/contact effects, Street Deal generation/resolution, debts processing, TurfLevelLogic, GameStateManager или UI.

M5 может определить orchestration hook для будущего модуля, но не должен поглощать M6–M13.
