PHASE CONTRACT:

Match:

- ровно 15 rounds;
- initial gameplay round — 1;
- round увеличивается только после завершённого Action без Street Deal либо после завершённого Street Deal;
- после round 15 Action переход идёт в Game Over без increment.

Transitions:

- valid completed Setup → Income;
- completed Income → Market;
- Market ждёт `ready_for_action == true` у всех players;
- completed Market → Action;
- Action остаётся Action, пока не завершены все players;
- completed Action rounds 4/8/12 → Street Deal;
- completed Action other rounds `< 15` → next round Income;
- completed Street Deal → next round Income;
- completed Action round 15 → Game Over;
- Game Over не продвигается дальше.

Action:

- order: `player_1`, `ai_1`, `ai_2`, `ai_3`;
- active player следует order;
- done players пропускаются при advancement;
- после последнего player active ID очищается;
- outside Action active ID пустой.

Skip action:

- проверяется, когда player становится active;
- `skip_next_action` сбрасывается;
- `action_done` становится true;
- hand и random state не изменяются;
- active player продвигается;
- controller не запускает combat/contract effects.

Resets:

- применяй только reset rules из owner schema/core-loop PRD;
- не очищай persistent match data;
- `used_deal_ids` сохраняется;
- per-round flags и contact usage сбрасываются только по documented timing;
- reset helpers не реализуют gameplay effects.

MUTATION CONTRACT:

Каждая mutating operation:

1. Проверяет input/preconditions.
2. Работает с `state.duplicate(true)`.
3. Применяет только transition-owned changes.
4. Валидирует resulting state.
5. Возвращает changed state только при success.
6. При failure возвращает original state unchanged.

Используй stable result shape проекта:

```gdscript
{
	"ok": true,
	"error": ValidationErrors.OK,
	"state": {},
	"log_entries": []
}
```

Не создавай ad-hoc error strings.

WINNER CONTRACT:

Base winner resolution:

1. Highest VP.
2. Highest Nal.
3. Highest total Status building VP value.
4. Highest number of Status buildings.
5. Stable order: `player_1`, `ai_1`, `ai_2`, `ai_3`.

Status tie-break values:

- stash = 1;
- workshop = 2;
- district_control = 3.

Эти значения используются только для tie-break и не добавляют VP.

Winner result должен соответствовать M4:

- `winner_id`;
- `final_scores`;
- `tie_break_used`;
- `tie_break_steps`;
- `turf_level_10_ai_win_applied`.
