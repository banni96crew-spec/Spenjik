ОГРАНИЧЕНИЯ:

- Не начинать M1 при проваленном M0 gate.
- Не выходить за пределы M1.
- Не реализовывать gameplay logic.
- Не реализовывать runtime state.
- Не создавать или изменять игровые правила.
- Не менять цены, эффекты, условия, формулы или баланс.
- Не переименовывать существующие protected IDs.
- Не добавлять undocumented IDs.
- Не добавлять ad-hoc validation error strings.
- Не объединять все domains в один чрезмерно крупный constants-файл.
- Не создавать альтернативный источник истины для одного ID.
- Не создавать `.tres` Resources.
- Не создавать Resource schemas или catalogs.
- Не реализовывать deterministic random.
- Не реализовывать GameStateManager.
- Не регистрировать gameplay Autoload.
- Не реализовывать economy, market, combat, roles, contracts, contacts, Street Deals, debts, Turf Levels или AI.
- Не создавать gameplay UI.
- Не добавлять React, TypeScript, JavaScript gameplay runtime, Tailwind, Zustand, Docker, backend или web routing.
- Не использовать `randf()`, `randi()`, `randomize()` или `RandomNumberGenerator` в gameplay logic.
- Каждый `.gd` файл должен быть меньше 250 строк.
- Использовать static typing там, где это практично.
- Не ослаблять тесты ради формального pass.
- Не заявлять о прохождении тестов без фактического запуска.
- Не удалять и не откатывать несвязанные пользовательские изменения.
- Не оставлять ambiguity markers без `OQ-*`.
- Не придумывать отсутствующие ID или error codes.

Если информации недостаточно или документы конфликтуют:

1. Останови только затронутую часть реализации.
2. Проверь существующие `OQ-*` и `FIX-*`.
3. Определи owner PRD спорного ID или error code.
4. Добавь новый `OQ-*` в `res://docs/prd/21_OPEN_QUESTIONS_AND_FIXES.md`, если вопрос ещё не зарегистрирован.
5. Не создавай временный ID или error code как скрытую догадку.
6. Продолжай только однозначно определённую часть M1.
