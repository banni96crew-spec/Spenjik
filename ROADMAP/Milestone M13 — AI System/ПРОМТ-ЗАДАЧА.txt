9. AI Market behavior

AI Market call is valid only if:

current phase is Market;
player is AI;
player has assigned valid AI profile;
state validates.

Market loop:

1. Read current market.
2. Build valid purchase candidates.
3. Score candidates.
4. Choose best candidate.
5. Buy through MarketLogic only.
6. After success, re-evaluate state and market.
7. Repeat until no normal/fallback purchase.
8. End market participation through phase-safe existing boundary.

AI can buy multiple cards in one Market.

Loop guard:

max iterations = available market card count + 1

If a candidate unexpectedly fails:

- do not corrupt state;
- exclude failed candidate for this loop;
- continue only if safe;
- otherwise fallback/end.

Normal candidate requires:

card exists in market;
Market validation succeeds;
final price preview succeeds;
player can afford;
nal - price >= profile.minimum_reserve_nal;
profile base score > 0.

Reserve does not change price and does not bypass validation.

Level 9

Use only:

TurfLevelLogic.get_ai_war_purchase_weight_multiplier(state, card_def)

Rules:

Level 9+;
human strictly leads all AI by VP;
candidate card is War;
score multiplier 1.2.

Do not duplicate Level 9 logic inside AI.

Purchase tie
unique highest score => no RNG;
equal highest final score => SeededPicker tie-break;
returned random stored in state.

Fallbacks:

In Market:

Fallback never bypasses:

Market validation;

After successful fallback purchase, loop re-evaluates.

11. Target scoring

Exclude attacker from targets.

A target is valid only if there is at least one legal attack option against it.

Formula:

vp lead * vpLead weight
+ available Nal * availableNal weight
+ missing active defenses * lowDefense weight
+ destructible Status count * destructibleBuildings weight
+ revenge flag * revenge weight
+ human flag * humanBias weight

Definitions:

VP lead: max(0, target.vp - attacker.vp);
available Nal: max(0, target.nal - protected_nal);
protected Nal comes from existing protected Nal API;
missing active defenses: inactive Cops + inactive Cartel + inactive Judge;
destructible status: stash + workshop + district_control;
revenge: target_id == attacker["last_attacked_by"];
human bias: target is human.

Scoring is read-only.

Target tie:

unique best => no RNG;
real tie => SeededPicker.
12. AI Action probability

At start of AI Action:

if hand has no War cards:
    end Action without RNG
else:
    roll exactly once via SeededRandom.next

Probability tag:

ai_%s_attack_probability_round_%s

Pass/fail:

roll <= profile.attack_probability => may attack
roll > profile.attack_probability => end Action

Failed roll:

random step is saved;
no attack;
no discard unless explicitly allowed by PRD fallback after passed roll;
hand remains unchanged;
end Action through phase-safe boundary.
13. AI Action loop

After passed probability roll:

1. Record count of War cards in hand at Action start.
2. Build attack options.
3. Prefer unblocked options.
4. Choose highest option score.
5. Tie => SeededPicker.
6. Resolve through CombatEngine only.
7. Re-evaluate hand and board.
8. Repeat until no unblocked option or loop limit.
9. Apply action fallback if needed.
10. End Action through phase-safe boundary.

Maximum resolved attacks:

<= War cards in hand at Action start

New War cards appearing during Action are not played until a future turn.

AI must not directly remove cards from hand except through CombatEngine discard API for discard fallback.

14. Attack options

Final option score:

card preference score + target score

Card preference comes from profile purchase scores.

Missing/zero score is allowed only for fallback candidate.

Each option must pass:

Combat validation;
combat preview;
mode requirements;
target requirements;
engine target requirements;
modifier requirements.

Rules:

Thug:
- no mode.

Bruiser:
- prefer valid unblocked destroy_stash;
- otherwise steal_nal.

Cleaner:
- prefer valid unblocked destroy_workshop;
- otherwise steal_nal.

Federal Raid:
- only destroy_district.

Saboteur:
- no mode;
- explicit engine target required.

Insider:
- only modifier for Thug.

Blocked preference:

if unblocked options exist, exclude blocked options;
blocked option allowed only if no unblocked options and fallback is attack_best_target;
blocked resolution still goes through CombatEngine and consumes card by M7 rules.

4. informant

No RNG.
