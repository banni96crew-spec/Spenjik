DETERMINISTIC RNG CONTRACT:

Все calls используют общий `state["random"]`.
Никаких отдельных seeds/streams.

Setup order:

1. `SeededPicker.pick_one` выбирает strong AI.
2. `SeededPicker.pick_unique` выбирает 3 profiles.
3. Returned random записывается после каждого call.

Setup расходует:

- 1 step на strong AI;
- 3 steps на profiles;
- всего 4 steps.

Probability:

- если в hand нет War cards, roll не выполняется;
- иначе один `SeededRandom.next` на AI Action;
- tag:
  `ai_%s_attack_probability_round_%s`;
- дополнительных probability rolls в этом turn нет.

Tie-break:

- purchase tie: +1 step только при real tie;
- target tie: +1 step только при real tie;
- attack-option tie: SeededPicker только при real tie;
- unique best не consume-ит tie RNG;
- Saboteur engine priority не использует RNG.

SETUP:

Required order:

1. Создать base players.
2. Выбрать strong AI.
3. Установить strong flags.
4. Выбрать 3 unique profiles.
5. Assign selected profiles в order `ai_1`, `ai_2`, `ai_3`.
6. Создать `ai_bosses`.
7. Применить Turf setup Levels 1/2.
8. Провалидировать state.

Same seed/state даёт identical setup.
Failed setup не мутирует players, bosses или random.

AI MARKET:

AI действует только:

- в Market phase;
- для valid AI player;
- с assigned valid profile.

Loop:

1. Получить current available card IDs.
2. Для каждой card вызвать MarketLogic validation.
3. Получить final price preview из PriceLogic.
4. Проверить reserve.
5. Получить base profile score.
6. Применить Level 9 multiplier к War score при необходимости.
7. Выбрать highest final score.
8. При tie использовать SeededPicker.
9. Купить через MarketLogic.
10. После success полностью re-evaluate state/market.
11. Повторять, пока есть normal/fallback purchase.
12. Завершить Market через phase-safe owner boundary.

Loop guard:

- максимум available market IDs + 1 iterations;
- unexpected failed candidate исключается без state corruption;
- infinite loop запрещён.

AI может купить несколько cards в один Market.
One-copy-per-ID-per-round и все limits сохраняются.

PURCHASE CANDIDATES:

Normal candidate требует:

- card доступна в market;
- MarketLogic validation succeeds;
- final price доступна;
- `nal - price >= minimum_reserve_nal`;
- base profile score > 0.

Missing score = 0.
Reserve не меняет цену и не bypass-ит affordability.

Level 9:

- если human strictly leads every AI by VP;
- только War candidate score ×1.2;
- price не меняется;
- Resource score не мутируется.

Candidate содержит player/card, base/final score, price,
reserve after purchase и applied modifiers.

PURCHASE TIE:

- unique highest score выбирается без RNG;
- equal highest final scores выбираются SeededPicker;
- returned random сохраняется;
- stable order не заменяет required random tie-break.

MARKET FALLBACKS:

- `end_phase`: завершить Market без purchase;
- `hold_nal`: завершить без purchase;
- `buy_cheapest_valid`: купить cheapest valid reserve-safe card;
- `discard_action_cards`: в Market завершить без purchase;
- `attack_best_target`: в Market завершить без purchase.

Cheapest price tie использует SeededPicker.
Fallback не bypass-ит MarketLogic validation/reserve.
После successful fallback purchase loop re-evaluates.

TARGET SCORING:

Исключи attacker из targets.
Каждый target должен быть legal для хотя бы одной attack option.

Формула:

```text
vp lead * vpLead weight
+ available Nal * availableNal weight
+ missing active defenses * lowDefense weight
+ destructible Status count * destructibleBuildings weight
+ revenge flag * revenge weight
+ human flag * humanBias weight
```

Feature definitions:

- VP lead: `max(0, target VP - attacker VP)`;
- available Nal: target Nal minus protected Nal, minimum 0;
- missing defenses: inactive Cops/Cartel/Judge, range 0..3;
- destructible: Stash + Workshop + District Control;
- revenge: target ID equals attacker `last_attacked_by`;
- human bias: target is human.

Protected Nal бери из M6 `PriceLogic`.
Scoring не мутирует state.
Target tie использует SeededPicker только при real tie.

AI ACTION PROBABILITY:

В начале active AI Action:

- если hand не содержит War, завершить Action без RNG;
- иначе выполнить ровно один probability roll;
- pass при `roll <= attack_probability`;
- fail при `roll > attack_probability`.

Failed roll:

- немедленно завершает Action;
- fallback не может заставить атаковать;
- random step сохраняется;
- War cards остаются в hand.

AI ACTION LOOP:

После passed roll AI может сыграть несколько War cards.

1. Зафиксировать число War cards в hand на Action start.
2. Построить valid attack options.
3. Предпочесть unblocked options.
4. Выбрать highest final option score.
5. При tie использовать SeededPicker.
6. Resolve через CombatEngine.
7. Re-evaluate hand и board.
8. Повторять до отсутствия unblocked option или loop limit.
9. Применить Action fallback при необходимости.
10. Завершить Action через phase-safe owner boundary.

Maximum attack resolutions не превышает стартовое число War cards.
Новые War cards, появившиеся в этом Action, не играются до будущего turn.
Unused cards остаются в hand, кроме explicit discard fallback.

ATTACK OPTION SCORE:

```text
final score = card preference score + target score
```

Card preference берётся из profile purchase scores.
Missing/zero score допускается только как fallback candidate.

Каждый option проходит:

- CombatEngine validation;
- read-only combat preview;
- mode/target/engine/modifier validation.

AI не мутирует hand или combat state напрямую.

CARD/MODE RULES:

- Thug: без mode.
- Bruiser: valid unblocked destroy Stash, иначе steal Nal.
- Cleaner: valid unblocked destroy Workshop, иначе steal Nal.
- Federal Raid: только `destroy_district`.
- Saboteur: без mode, но с explicit engine target.
- Insider: только modifier для Thug.

Blocked destroy не превращается в invalid hidden effect:
AI строит отдельный legal steal option через Combat API.

SABOTEUR:

Engine priority строго:

1. Brothel.
2. Laundry.
3. Accountant.
4. Informant.

Выбери первый owned valid engine.
RNG не использовать.
Без valid engine Saboteur option не создаётся.

INSIDER:

Использовать только если:

- AI имеет Thug и Insider;
- target имеет active Cops;
- Thug без Insider blocked;
- Thug с Insider unblocked.

Insider нельзя использовать primary или с другими War cards.

BLOCKED OPTIONS:

- при наличии unblocked options blocked options исключаются;
- blocked option разрешён только если unblocked нет
  и fallback = `attack_best_target`;
- probability roll уже должен быть passed;
- resolution всё равно идёт через CombatEngine и расходует card по M7.

ACTION FALLBACKS:

- `end_phase`: завершить Action;
- `hold_nal`: завершить Action;
- `buy_cheapest_valid`: в Action завершить;
- `discard_action_cards`: discard War cards без valid unblocked option
  через CombatEngine discard API, затем завершить;
- `attack_best_target`: разрешить best blocked valid option
  только после passed roll и при отсутствии unblocked options.

Fallback не bypass-ит validation и probability.

PHASE/PLAYER RULES:

- Market AI call для human fails.
- Action AI call для human fails.
- Action player должен совпадать с `active_action_player_id`.
- `skip_next_action` обрабатывает phase owner, AI его не отменяет.
- AI не вызывает direct phase transition.
- До M14 используй существующий phase-safe internal completion API.

AI не получает roles, contracts, contacts и не выбирает Street Deals.
Explicit M10 side effects на AI остаются допустимыми.
