UI OWNERSHIP:

UI может:

- render safe views;
- хранить selection/hover/expanded state;
- собирать payload;
- проверять очевидную completeness payload;
- запрашивать previews/disabled reasons;
- вызывать GameStateManager;
- показывать errors и logs;
- переключать screens/panels;
- управлять focus, scroll и presentation locks.

UI не может:

- вычислять prices/affordability;
- проверять market availability;
- вычислять attack block/effect;
- применять card/role/contract/contact/Street Deal effects;
- менять Nal, VP, hands, buildings, debts или logs;
- писать phase/readiness/action flags;
- выбирать AI action;
- consume-ить gameplay random;
- парсить descriptions как rules;
- хранить копию authoritative gameplay state.

SCENE COMPOSITION:

`Main.tscn`:

- project entry scene;
- создаёт/показывает GameRoot;
- не владеет gameplay state;
- не содержит gameplay orchestration.

`GameRoot.tscn`:

- screen host/coordinator;
- содержит SetupScreen, GameScreen и GameOverScreen;
- переключает visibility по active game/phase/signals;
- подписывается на GameStateManager signals один раз;
- не дублирует panel gameplay behavior.

Screen switching:

- no active game -> SetupScreen;
- successful `game_started` -> GameScreen;
- successful `game_ended`/game_over view -> GameOverScreen;
- New Game очищает/перезапускает только через documented facade flow;
- Exit/Main Menu не реализует save/load.

SETUP SCREEN:

Собирает:

- non-empty seed;
- Turf Level 0..10 из `get_available_turf_levels()`;
- role из `get_available_roles()`;
- one contract из deterministic offer.

Flow:

1. User выбирает seed, Turf и role.
2. UI вызывает `generate_contract_offers(config)`.
3. Показывает returned offer IDs/definitions.
4. User выбирает one contract.
5. UI вызывает `start_new_game(config)`.
6. Success открывает GameScreen.
7. Failure показывает readable error и остаётся в Setup.

UI может проверить только completeness:

- seed не пуст;
- role/Turf/contract выбраны.

Validity/offer membership проверяет GameStateManager.
Не генерируй seed через random/time без явной PRD.

GAME SCREEN:

Обязательно показывает:

- round и current phase;
- active action player;
- все 4 PlayerBoard;
- human-relevant controls;
- MarketPanel;
- ActionPanel;
- StreetDealPanel;
- ContactPanel;
- ContractPanel;
- GameLogPanel;
- waiting/AI activity state.

Panels показываются/disabled по phase/view, но UI не меняет phase.
Phase visibility не является заменой owner validation.

PLAYER BOARD:

Render для каждого player:

- stable player identity/display label;
- human/AI/strong AI/profile indicators, если доступны view;
- Nal/VP через NalVpDisplay;
- Engine, Status и Defense summaries;
- War hand только в разрешённом human-safe representation;
- readiness/action status;
- relevant contact/contract badges для human.

Не раскрывай hidden information, если safe view её не предоставляет.
Не читай raw active state ради удобства.

MARKET PANEL:

- получает data через `get_market_view(human_id)`;
- отображает available cards через CardView;
- при selection вызывает price preview и disabled reason;
- показывает final price/modifiers из preview;
- Buy вызывает `buy_card`;
- End Market вызывает `end_market_for_player`;
- Rebuild вызывает `rebuild_district_control`;
- после любого result refresh-ится из facade.

UI не subtract-ит Nal, не place-ит card и не вычисляет requirements.

ACTION PANEL:

Local state:

- selected card ID;
- selected target ID;
- selected mode;
- selected engine target ID;
- selected modifier IDs.

Flow:

1. Покажи War cards из safe view.
2. Получи targets через `get_valid_targets`.
3. Получи engine targets для Saboteur.
4. Собери exact combat payload.
5. Получи disabled reason и `get_combat_preview`.
6. Покажи target/mode/block/effect/cards-consumed preview.
7. Execute вызывает `execute_attack`.
8. Discard вызывает `discard_war_card`.
9. End Action вызывает `end_action_for_player`.

Payload fields:

- `attacker_id`;
- `target_id`;
- `card_id`;
- `mode`;
- `modifiers`;
- `engine_target_card_id`.

Completeness behavior:

- Bruiser/Cleaner требуют selected documented mode.
- Federal Raid требует `destroy_district`.
- Saboteur требует explicit engine target.
- Insider доступен только как modifier option для Thug.

Не реализуй effect/defense rules в UI.
Какие options действительно legal, определяют selectors.

STREET DEAL PANEL:

- видим только в relevant phase/view;
- получает data через `get_street_deal_view`;
- показывает title/description и Option A/B;
- запрашивает disabled reason для каждой option;
- вызывает `select_street_deal` с exact payload;
- не применяет effects и не создаёт debts;
- после success показывает ContactPanel при pending offer.

Payload:

- human `player_id`;
- current `deal_id`;
- selected `"A"` или `"B"`.

CONTACT PANEL:

- получает pending offer через `get_contact_offer`;
- owned state через `get_contact_state`;
- disabled reason через `get_contact_disabled_reason`;
- selection вызывает `select_contact`;
- manual activation показывается только если actual M14/owner contract
  предоставляет применимый active contact.

Не предлагай manual pre-activation Street Medic:
его MVP use case идёт через DebtLogic hook.

UI не unlock-ит contact и не меняет pending offer.

CONTRACT PANEL:

- получает runtime через `get_contract_state`;
- показывает ID/title/progress/deadline/status/reward view;
- Claim availability получает через disabled reason selector;
- Claim вызывает `claim_contract`;
- не проверяет condition и не начисляет reward.

GAME LOG PANEL:

- render-ит logs из safe view;
- корректно обрабатывает empty logs;
- позволяет scroll/expand entries;
- не создаёт gameplay log entries;
- не добавляет wall-clock gameplay timestamps.

GAME OVER SCREEN:

Показывает:

- winner;
- final VP/Nal;
- game result/tie-break summary;
- contract result;
- Turf Level;

WIDGETS:

`CardView`:

- render-ит card identity/display data, type, price/preview и state;
- emits UI selection/request signal;
- не validates gameplay.

`DefenseBadges`:

- render-ит Cops/Cartel/Judge status из view;

- render-ит Nal/VP;
- не изменяет values.

- получает stable code;
- использует ErrorTextMap;

`UIViewFormatters`:

- pure presentation formatting;
- no gameplay decisions;
- no state mutation.

SIGNALS И REFRESH:

Подпишись на:

- `state_changed`;
- `action_failed`;
- `phase_changed`;
- `game_started`;
- `game_ended`.

После каждого mutator:

1. Inspect `ok/error`.
2. Refresh authoritative view на success.
3. Show mapped error и также refresh на failure.
4. Не patch-ить labels по предполагаемому result.
5. Не делать automatic retry.

Предотвращай duplicate signal connections и stale callbacks.
Scene teardown/reload не должен удваивать refresh.

AI VISIBILITY:

- AI actions могут выполняться автоматически owner flow;
- UI показывает waiting/busy state;
- human command buttons disabled во время active AI resolution;
- purchases/attacks/blocked results показываются через logs/summaries;
- UI не запускает AI вне phase owner;
- AI debug buttons допустимы только явно dev/debug и не обязательны.

ERROR TEXT MAP:

- key = stable ValidationErrors code;
- value = readable presentation text;
- `ValidationErrors.OK` отображается как available/empty helper text;
- unknown code отображается безопасно, но не подменяется другим code;
- mapping не содержит validation conditions;
- localization framework не требуется в M16.
