# Проект: The Turf / Передел

## О чём продукт

The Turf / «Передел» — desktop single-player card/board mini-game
внутри визуальной новеллы.

В матче участвуют один human player и три local AI.
Партия длится 15 раундов и проходит от Setup до Game Over.

Текущий этап: Milestone M16 — UI / UX.
Он выполняется строго после завершённых M0–M15.

## Стек

- Godot 4.6.2 stable.
- GDScript со статической типизацией, где это практично.
- Godot Control nodes, Containers и Theme.
- GUT.
- `.tres` Resources для static definitions.
- Dictionary snapshots для runtime state.
- GameStateManager как Autoload facade.
- Windows/Linux desktop first.
- Без web stack.

## Архитектура

- Logic-first, data-driven.
- UI является view/input layer, а не gameplay owner.
- Весь gameplay state принадлежит GameStateManager.
- UI получает data через `get_view()` и public selectors.
- UI вызывает gameplay mutators только через GameStateManager.
- UI не вызывает low-level logic modules для mutation.
- UI хранит только temporary selection/presentation state.
- После любого command UI refresh-ится из authoritative view.
- Disabled reasons приходят из facade selectors.
- Complex actions используют read-only previews.
- Stable error codes переводятся в readable UI text.
- Signals управляют refresh и screen/phase visibility.
- Gameplay random в UI запрещён.
- Resources и snapshots не мутируются UI scripts.
- M15 integration/replay flow является уже проверенной основой.

  Street Deal, contact, AI APIs, selectors и signals;

## Договорённости по коду

- Каждый UI `.gd` файл меньше 250 строк.
- Gameplay mutations идут только через GameStateManager.
- UI не читает `GameStateManager.state` напрямую.
- UI не меняет Nal, VP, cards, logs, flags или phase fields.
- UI не рассчитывает price, block, rewards или AI decisions.
- UI validation ограничена completeness user input.
- Button-based core actions обязательны; drag-and-drop не требуется.
- Desktop landscape baseline: минимум 1280x720, target 1920x1080.
- При недостатке места core controls остаются доступны через Containers/Scroll.
- IDs передаются без преобразования display text.
- ErrorTextMap форматирует только known stable codes.
- UI helpers не содержат gameplay logic.
- Selection очищается при success, cancel или relevant phase/state change.
- AI activity показывается через summaries/logs, без обязательных animations.
- Existing Theme/assets переиспользуются; отсутствие art direction
  не является разрешением изобретать gameplay meaning.
- Все resolved FIX-001–FIX-030 обязательны.
- Неясность фиксируется как `OQ-*`.
- Ориентиры длины промтов приблизительные.

## Что НЕ входит в текущий milestone

- Изменение gameplay rules, state schema, balance или AI behavior.
- Прямые вызовы MarketLogic, CombatEngine и других owner mutators.
- Gameplay data store внутри UI.
- Автоматические скрытые покупки, атаки, claims или choices.
- Полноценная локализация, accessibility overhaul и art production.
- Сложные AI animations и cinematic transitions.
- Save/load, campaign persistence или networking.
- M17 production polish, analytics и final hardening.
- Mobile-first layout или обязательный drag-and-drop.
- React, TypeScript, Tailwind, Zustand, Docker, backend и routing.
