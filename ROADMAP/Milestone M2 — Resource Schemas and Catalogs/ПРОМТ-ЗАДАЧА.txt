КОНТЕКСТ ПРОЕКТА:

“The Turf” / “Передел” — desktop single-player card/board mini-game внутри визуальной новеллы.

Целевой стек:

- Godot 4.6.2 stable;
- GDScript;
- GUT;
- `.tres` Resources для static gameplay data;
- domain catalogs для доступа к Resources;
- Dictionary snapshots для будущего runtime state;
- `GameStateManager.gd` как будущий gameplay facade;
- Windows и Linux как первые export targets;
- без web stack.

Архитектура:

UI → GameStateManager → logic modules → catalogs/resources/constants

Resource layer должен быть:

- data-only;
- типизированным;
- неизменяемым во время gameplay;
- доступным через catalogs;
- согласованным с constants из M1;
- не содержащим gameplay resolution logic.

Предполагается, что M0 и M1 завершены. Перед началом M2 проверь это фактически.

ОГРАНИЧЕНИЯ:

- Не начинать M2 при проваленном M0 или M1 gate.
- Не выходить за пределы M2.
- Не реализовывать gameplay logic.
- Не реализовывать эффекты, указанные в Resources.
- Не реализовывать runtime state.
- Не хранить ownership, cooldown, progress, market state, debts или random state в Resources.
- Не изменять constants из M1 без прямого требования owner PRD.
- Не менять ID, цены, значения, эффекты, условия или баланс.
- Не добавлять undocumented Resource fields.
- Не добавлять undocumented `.tres`.
- Не создавать fallback Resource для неизвестного ID.
- Не использовать display descriptions как машиночитаемые правила.
- Не мутировать Resources после загрузки.
- Не загружать `.tres` ad hoc из будущих gameplay modules.
- Не создавать один mega-catalog для всех доменов.
- Не реализовывать deterministic random.
- Не реализовывать GameStateFactory, GameStateValidator или GameStateManager.
- Не реализовывать economy, combat, roles logic, contracts logic, Street Deals logic, debts, contacts logic, Turf Levels logic или AI.
- Не создавать gameplay scenes или UI.
- Не добавлять React, TypeScript, Tailwind, Zustand, Docker, backend или web routing.
- Не использовать `randf()`, `randi()`, `randomize()` или `RandomNumberGenerator` в gameplay logic.
- Каждый `.gd` файл должен быть меньше 250 строк.
- Использовать static typing там, где это практично.
- Не ослаблять тесты ради формального pass.
- Не заявлять о прохождении тестов без фактического запуска.
- Не удалять и не откатывать несвязанные пользовательские изменения.
- Не оставлять ambiguity markers без `OQ-*`.
- Не придумывать отсутствующие schema fields или Resource values.

Если информации недостаточно или PRD конфликтуют:

1. Останови только затронутый Resource domain.
2. Проверь существующие `OQ-*` и `FIX-*`.
3. Определи owner PRD спорного поля или значения.
4. Добавь новый `OQ-*` в `res://docs/prd/21_OPEN_QUESTIONS_AND_FIXES.md`, если вопрос ещё не зарегистрирован.
5. Не добавляй временное значение или скрытый fallback.
6. Продолжай только однозначно определённые части M2.
