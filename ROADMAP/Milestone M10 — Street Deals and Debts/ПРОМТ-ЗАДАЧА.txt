GENERATION:

Street Deal генерируется только:

- в `PhaseIds.STREET_DEAL`;
- в раундах 4, 8 и 12;
- после завершённого Action phase.

При входе в новую Street Deal phase:

1. Сбросить current offer fields и human choice.
2. Не очищать `used_deal_ids`.
3. Построить eligible weighted pool.
4. Применить `min_round`.
5. Исключить used deals.
6. Исключить `loan_shark` при active human debt.
7. Выбрать один deal через `SeededPicker.pick_weighted`.
8. Использовать tag `street_deal_round_%s`.
9. Записать returned random state и current deal.
10. Установить `offered_this_round = true`.

Weighted generation расходует ровно один RNG step.
No eligible deal возвращает no-offer result без effect mutation.
Active debt не исключает никакие deals, кроме `loan_shark`.

SELECTION И VALIDATION:

Payload:

```gdscript
{
    "player_id": "player_1",
    "deal_id": "dirty_tip",
    "option_id": "A"
}
```

Валидируй:

- current phase;
- round;
- current deal существует и совпадает с payload;
- player является human;
- option равна `A` или `B`;
- human ещё не выбирал в этой phase;
- upfront Nal достаточно;
- Loan Shark не имеет active debt;
- Inside Contact A имеет доступный handoff path.

Ошибки используй из constants:

- `INVALID_PHASE`;
- `INVALID_TARGET`;
- `NOT_ENOUGH_NAL`;
- `ACTIVE_DEBT_EXISTS`;
- `STREET_DEAL_CHOICE_UNAVAILABLE`;
- специализированные IDs, если они существуют.

Failed validation не мутирует state, RNG, hand, Nal, VP,
used deals, choices, debts, modifiers или logs.

После success:

- применить effects в указанном порядке;
- записать human choice;
- добавить deal в `used_deal_ids` ровно один раз;
- вызвать ContractLogic hook после state changes;
- создать structured log/result;
- очистить current deal, кроме необходимого pending handoff;
- не продвигать phase самостоятельно.

STREET DEAL EFFECTS:

Loan Shark, доступен с round 8 и без active debt.

Option A:

- human Nal +10;
- создать debt amount 12;
- deadline = current round +2;
- overdue penalty: lose all Nal и -1 VP.

Option B:

- human Nal +5;
- создать debt amount 6;
- deadline = current round +2;
- overdue penalty: -1 VP.

Debt ID должен быть стабильным, например:

- `loan_shark_round_8_A`.

Dirty Tip, доступен с round 4.

Option A:

- оплатить 3 Nal, либо 4 при Turf 8+;
- добавить один Bruiser в human hand.

Option B:

- human Nal +3;
- один deterministic random AI получает один Thug;
- выбирать только из `ai_1`, `ai_2`, `ai_3`;
- использовать `SeededPicker` и обновить random state;
- расходовать ровно один дополнительный RNG step.

Cheap Protection, доступен с round 4.

Option A добавляет temporary modifier:

- next Defense card price delta -2;
- minimum price 1;
- expires at next matching purchase;
- consumed false.

Option B:

- human Nal +2;
- temporary next War card price delta +1;
- expires at next matching purchase;
- consumed false.

PriceLogic применяет и consume-ит modifier.
StreetDealLogic не реализует purchase.
При duplicate unconsumed modifier используй existing stacking convention;
не изобретай новое правило без проверки PRD/repository.

Black Market Cache, доступен с round 4.

Option A:

- human Nal +6.

Option B:

- оплатить 6 Nal, либо 7 при Turf 8+;
- human VP +1.

Inside Contact, доступен с round 8.

Option A:

- запросить future offer из 2 available contacts;
- создать structured pending handoff;
- не unlock-ить contact напрямую.

M11 ещё не реализован:

- не создавай fake `ContactLogic`;
- используй существующий integration convention, если он уже есть;
- иначе создай только безопасный pending request contract,
  который M11 сможет обработать;
- фактическое offer generation/selection принадлежит M11.

Option B:

- human Nal +4.

Risky Contract, доступен только с round 12.

Option A:

- оплатить 3 Nal, либо 4 при Turf 8+;
- human VP +1.

Option B:

- human Nal +5;
- richest AI Nal +1;
- highest Nal wins;
- tie-break строго `ai_1`, затем `ai_2`, затем `ai_3`;
- RNG не использовать.

NAL/VP clamp:

- Nal и VP не могут стать меньше 0;
- paid option сначала проверяет достаточность Nal;
- partial effect при failed payment запрещён.

DEBT PROCESSING:

Debts обрабатываются для игрока в stable array order во время Income:

1. Income gain.
2. Cops upkeep.
3. Debt processing.
4. Contract state-change hook.

Для каждой active debt:

- если Nal >= amount due: auto-repay;
- иначе при `round <= deadline`: оставить active;
- иначе при `round > deadline`: применить penalty и закрыть debt.

Auto-repay:

- списывает exact amount due;
- `repaid = true`;
- `repaid_round = current round`;
- создаёт result/log.

Overdue penalty:

- Option A debt может установить Nal = 0;
- применяет VP delta и clamp к 0;
- `repaid = true`;
- `penalty_applied_round = current round`;
- повторно не применяется.

Repaid debts пропускаются.
Manual repayment не добавляй.
