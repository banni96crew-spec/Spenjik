# Проект: The Turf / Передел

## О чём продукт

“The Turf” / “Передел” — desktop single-player card/board mini-game внутри визуальной новеллы.
Разработка идёт строго по milestone-порядку:

M0 Project Bootstrap
→ M1 Constants and IDs
→ M2 Resource Schemas and Catalogs
→ M3 Deterministic Random
→ последующие gameplay-модули

Игровое ядро должно быть детерминированным, тестируемым без UI и пригодным для replay. Одинаковые seed, исходное состояние, решения и порядок random-вызовов должны давать одинаковый результат.

Документы в `res://docs/prd/` — источник истины. Не придумывай отсутствующие алгоритмы, API, state fields или gameplay rules. При неоднозначности проверь `21_OPEN_QUESTIONS_AND_FIXES.md`; если решения нет, останови затронутую часть и зарегистрируй `OQ-*`.

## Стек

- Godot 4.6.2 stable.
- GDScript и GUT.
- `.tres` Resources для статических данных.
- Dictionary snapshots для будущего runtime state.
- Windows и Linux как первые export targets.
- Без web stack.

## Архитектура

Будущее направление зависимостей:

UI → GameStateManager → logic modules → catalogs/resources/constants

Gameplay random принадлежит только:

- `res://logic/random/SeededRandom.gd`
- `res://logic/random/SeededPicker.gd`

Все будущие consumers используют единый `state["random"]`. Поле `random["step"]` — единственный счётчик gameplay random.
RandomState:

```gdscript
{
	"seed": String,
	"step": int,
	"last_random_tag": String,
	"random_history_enabled": bool
}
```

При включённой debug history разрешено поле `history: Array[Dictionary]`. Gameplay от history не зависит.

Критические invariants:

- Required algorithm: `cyrb53` + `mulberry32`.
- `seeded_random(seed, step)` — pure function.
- Один draw потребляет ровно один step.
- Seed не изменяется; tag не влияет на value.
- Failed validation не изменяет random state и не потребляет step.
- Picker не мутирует input arrays.
- Preview и selector functions не потребляют random.
- Winner resolution не использует random.
- Каждый `.gd` файл содержит менее 250 строк.

Запрещены в gameplay code:

- `randf()`, `randi()`, `randomize()`;
- `RandomNumberGenerator`;
- system time и OS random.
