КРИТИЧЕСКИЕ INVARIANTS:

- `seeded_random` использует `cyrb53` + `mulberry32`.
- Same seed + same step всегда дают same value.
- Value находится в диапазоне `0.0 <= value < 1.0`.
- `seeded_random` не мутирует state.
- `create_random_state` начинает с `step == 0`.
- `next` потребляет ровно 1 step, сохраняет seed и обновляет tag.
- Tag не участвует в hash и не влияет на value.
- History пишется только при `random_history_enabled == true`.
- Gameplay не зависит от history.
- `roll_d6_pair` потребляет ровно 2 steps и возвращает dice 1–6.
- Integer index: `floor(value * count)` с defensive clamp.
- Picker использует только `SeededRandom.next`.
- Failed validation потребляет 0 steps и возвращает исходный random state.
- Input arrays не мутируются.

PICKER CONTRACT:

`pick_one`:
- empty input → failure, 0 steps;
- valid input → selected item, 1 step.

`pick_unique`:
- `count <= 0` → success с empty selection, 0 steps;
- выбирает до `count` unique items;
- count больше pool → выбирает весь pool;
- потребляет 1 step на выбранный item;
- работает с копией pool.

`pick_weighted`:
- игнорирует `weight <= 0`;
- нет positive weights → failure, 0 steps;
- valid selection → 1 step;
- threshold равен `value * total_positive_weight`;
- stable input order определяет boundaries;
- input не сортируется.

`pick_best_tie`:
- empty list → failure, 0 steps;
- один item → success, 0 steps;
- несколько items → `pick_one`, 1 step.

Picker result shape содержит:

- `ok`, `error`;
- `selected`, `selected_items`;
- `steps_used`, `random`.

Используй только `ValidationErrors.OK` и `ValidationErrors.REQUIREMENT_NOT_MET`.

МИНИМАЛЬНЫЕ ТЕСТЫ:

SeededRandom:

- reproducibility для same seed/step;
- range `[0.0, 1.0)`;
- representative different seed/step samples;
- locked sample vectors;
- корректная initial RandomState shape;
- exact step и tag behavior;
- tag не влияет на value;
- history disabled/enabled.

Dice:

- два dice 1–6;
- корректные `sum` и `is_double`;
- ровно 2 steps;
- reproducibility из одинакового starting state.

SeededPicker:

- success, failure и exact steps всех методов;
- отсутствие duplicates;
- count 0 и count больше pool;
- non-positive weights;
- single-item и real tie;
- input arrays не мутируются;
- failure сохраняет random state.

Static scan:

- обнаруживает `randf(`, `randi(`, `randomize(` и `RandomNumberGenerator`;
- сканирует gameplay paths;
- исключает PRD и собственные test patterns без ослабления gameplay ban.
