# ПРОМТ-КОНТЕКСТ — M16.2 Card Visual System

## Проект: The Turf / Передел

### О чём продукт

The Turf / «Передел» — desktop single-player card/board mini-game внутри визуальной новеллы.

В матче участвуют:

* 1 human player;
* 3 local AI;
* партия длится 15 раундов;
* финал — Game Over.

Текущий этап: **Milestone M16.2 — Card Visual System**.

Это дополнительный UI-polish milestone после M16.1 и до M16.3/M17.

Цель M16.2 — довести карточный UI до единой визуальной системы на основе 4 утверждённых reference-карт:

* Engine;
* Status;
* War;
* Defense.

M16.2 не меняет gameplay rules, card effects, balance, deterministic replay, public facade contracts или state schema.

## Архитектура

Dependency direction:

`UI -> GameStateManager -> logic -> catalogs/resources/constants`

Правила:

* UI является view/input layer.
* Runtime state принадлежит GameStateManager.
* UI вызывает gameplay только через GameStateManager.
* UI не читает и не мутирует `GameStateManager.state`.
* CardView является presentation widget.
* CardView не владеет gameplay rules.
* CardView не парсит `effect_summary` как правила.
* CardView отображает уже готовые presentation dictionaries.
* CardView остаётся template-based.
* Full-card PNG не является основным способом отображения карты.
* Gameplay modules не зависят от UI scenes.
* M15 replay determinism и M16/M16.1 UI boundaries должны остаться зелёными.

## Договорённости по коду

* M16.2 не меняет gameplay rules.
* Не менять card effects, prices, IDs, formulas, phase order или AI policy.
* Не менять deterministic random.
* Не менять state schema.
* Не менять public GameStateManager contracts без подтверждённого blocker.
* Не добавлять save/load, persistence, campaign или profiles.
* Не читать raw `GameStateManager.state`.
* Не вызывать low-level owner modules из UI.
* Не использовать full-card PNG как основной renderer.
* Не делать special-case по card IDs вроде `laundry`, `stash`, `thug`, `cops`.
* Различие внешнего вида должно идти от `card_type`, а не от конкретной карты.
* Не парсить `effect_summary` как правила.
* Не изобретать новые gameplay icons со смыслом, которого нет в PRD.
* Все icons/markers являются presentation-only.
* Каждый `.gd` файл строго меньше 250 строк.
* Static scans нельзя ослаблять.
* UI boundary tests должны оставаться зелёными.
