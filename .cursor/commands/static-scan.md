# Статический анализ проекта

1. В терминале выполни следующие поисковые команды (примеры для Linux/macOS):
   - Проверка запрета случайных API:
     ```bash
     rg -n "\b(randf|randi|randomize|RandomNumberGenerator)\b" --glob "*.gd" res://logic
     ```
   - Поиск нежелательных импортов UI в логике:
     ```bash
     rg -n "res://scenes/ui" --glob "*.gd" res://logic
     ```
   - Проверка длины файлов: просмотр файлов `.gd`, превышающих 250 строк:
     ```bash
     find res://logic -name "*.gd" -exec awk 'NR==251{print FILENAME; exit}' {} \;
     ```
2. Если найдено нарушение, зафиксируй путь и причину, исправь код (перенеси случайность в `SeededRandom`, удаляй зависимость от UI, разбивай длинный файл). Не меняй правила игры.
3. Для Windows используй аналоги `grep`/`awk` (можно установить `ripgrep` и `gawk` или выполнить поиск средствами IDE).
